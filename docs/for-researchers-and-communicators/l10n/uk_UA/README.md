# Для дослідників і наукових комунікаторів

Статус: draft

Ontoverse може бути цікавим як приклад побудови pre-formal framework: як неінституційний автор організовує концептуальні інтуїції, аналогії, термінологію та спекулятивні структури до їх математичної формалізації або відкидання.

Проєкт не подається як доказ. Він подається як матеріал для перевірки, критики, порівняння та можливої формалізації.

## Переклади

- [English](../../)
- Українська

## Чим є Ontoverse

Ontoverse є самостійно розробленим концептуальним фреймворком для організації реальності, понять і можливих прихованих зв’язків між ними.

Він структурований як дослідницький блокнот, що містить:

- визначення;
- концептуальні моделі;
- діаграми;
- гіпотези;
- аналогії;
- відкриті проблеми;
- порівняння з наявними науковими та філософськими концепціями.

## Чим Ontoverse не є

Наразі Ontoverse не заявляється як:

- завершена фізична теорія;
- заміна квантовій механіці або теорії відносності;
- доказ прихованих змінних;
- доказ багатосвітової чи будь-якої іншої інтерпретації;
- новий експериментально підтверджений закон;
- містичне твердження чи одкровення.

## Корисні питання для перевірки

Продуктивна перевірка Ontoverse має запитувати:

- Що потрібно визначити математично?
- Які терміни дублюють наявні концепції?
- Які аналогії корисні, а які вводять в оману?
- Які частини можна відкинути одразу?
- Які частини є лише схемами найменування?
- Які частини містять потенційно корисну абстракцію?
- Яка наявна література найближча до кожної ідеї?
- Що могло б вважатися відмінним прогнозом, якщо такий узагалі є?

## Комунікаційний ракурс

Безпечний ракурс для публічної комунікації — не «хтось довів нову теорію».

Кращий ракурс:

```text
Чи може самостійно розроблений концептуальний фреймворк виявити корисну структуру до того, як наука матиме для неї формальну назву?
```

Або:

```text
Як спекулятивні інтуїції стають доступними для перевірки, критики та зрештою формалізації ідеями?
```

## Бажана критична позиція

До Ontoverse слід підходити з конструктивним скепсисом.

Мета не в тому, щоб захищати кожен термін. Мета — відокремити:

- корисні абстракції;
- оманливі аналогії;
- термінологію, яку слід перейменувати;
- ідеї, уже охоплені наявними галузями;
- невирішені частини, що потребують формальної роботи;
- частини, які слід відкинути.
