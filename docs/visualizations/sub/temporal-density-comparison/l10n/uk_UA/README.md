# Порівняння темпоральної щільності

Статус: draft

![Порівняння темпоральної щільності](../../../../assets/uk_UA/diagrams/temporal-density-comparison.svg)

Ця діаграма пояснює концепцію **temporal density** (темпоральна щільність) в Ontoverse, порівнюючи дві історичні траєкторії в межах одного **frontal-time interval** (інтервалу фронтального часу).

## Переклади

- [English](../../)
- Українська — поточна

## Що показує діаграма

Обидві траєкторії обмежені тими самими двома рубіновими frontal-time planes. Це означає, що їх оцінюють у межах одного frontal-time interval.

Верхня траєкторія має **низьку temporal density**:

- менше event-nodes;
- довші проміжки між значущими переходами;
- менше локальних можливостей розгалуження.

Нижня траєкторія має **високу temporal density**:

- більше event-nodes;
- коротші проміжки між значущими переходами;
- більше локальних можливостей розгалуження.

## Інтерпретація

В Ontoverse local time не розглядається лише як зовнішня координата. Він повʼязаний із накопиченням значущих event-nodes уздовж траєкторії.

Концептуальний вираз:

```text
temporal density ~ event-nodes / frontal-time interval
```

Це ще не фізичне рівняння. Це візуальне й концептуальне визначення, яке потребує майбутньої формалізації.

## Без future-side content

Траєкторії зупиняються на правій frontal-time plane.

Це навмисно: площина представляє межу теперішнього в показаному зрізі моделі. Вузли та реалізовані шляхи за цією площиною означали б майбутні події, яких ще немає в показаному model slice.

## Роль у документації

Використовуйте цю візуалізацію для пояснення:

- frontal time;
- local time;
- temporal density;
- накопичення event-nodes;
- чому різні історії можуть мати різну кількість змін у межах одного frontal-time interval.
