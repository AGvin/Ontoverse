# Контрольний список для перевірки

Статус: draft

Використовуйте цей контрольний список перед перенесенням дослідницького матеріалу Ontoverse до стабільної документації або перед публікацією пояснень для широкої аудиторії.

## Переклади

- [English](../../)
- Українська

## Межі тверджень

- Чи сторінка явно вказує свій статус?
- Чи сторінка не подає спекулятивні компоненти моделі як усталену науку?
- Чи сторінка не натякає на доказ, підтвердження або завершену теорію?
- Чи зрозуміло, що саме стверджується, а що лише досліджується?
- Чи скептичний читач може зрозуміти, чого сторінка не стверджує?

## Якість концепту

- Чи key terms визначені перед активним використанням?
- Чи proposed terms позначені як proposed Ontoverse terminology?
- Чи assumptions явні?
- Чи visible weak points та unresolved issues?
- Чи open formalization tasks перелічені там, де це доречно?

## Зв’язок із наявними ідеями

- Чи сторінка відокремлює усталене підґрунтя від інтерпретації Ontoverse?
- Чи пояснює вона, є концепція аналогією, гіпотезою чи запропонованим компонентом моделі?
- Чи уникає вона тверджень про еквівалентність із наявною фізикою, якщо таку еквівалентність не продемонстровано?
- Чи розміщені зовнішні наукові посилання або джерела натхнення поруч із твердженнями, що на них спираються?

## Публічне формулювання

- Чи сторінка відповідає позиціонуванню Ontoverse як самостійно розробленого концептуального фреймворку?
- Чи вона уникає містичних формулювань, непідкріпленої впевненості та мови грандіозної теорії?
- Чи вона зберігає відмінність між корисною спекуляцією та непідкріпленим твердженням?
- Чи вона полегшує критику, а не ускладнює її?

## Правило просування

Переносьте матеріал із `notes/` до `framework/`, `models/`, `glossary/` або на публічні сторінки лише тоді, коли термінологія та межі достатньо стабільні для повторного використання.

У разі сумнівів залишайте матеріал у `notes/` і позначайте його як дослідницький.
