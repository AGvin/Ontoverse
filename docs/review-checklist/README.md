# Review Checklist

Status: draft

Use this checklist before promoting exploratory Ontoverse material into stable documentation or before publishing public-facing explanations.

## Claim Boundaries

- Does the page clearly state its status?
- Does the page avoid presenting speculative model components as established science?
- Does the page avoid implying proof, confirmation, or a completed theory?
- Does the page clearly separate what is claimed from what is only being explored?
- Could a skeptical reader understand what is not being claimed?

## Concept Quality

- Are key terms defined before they are used heavily?
- Are proposed terms marked as proposed Ontoverse terminology?
- Are assumptions explicit?
- Are weak points or unresolved issues visible?
- Are open formalization tasks listed where appropriate?

## Relation to Existing Ideas

- Does the page separate established background from Ontoverse interpretation?
- Does it explain whether a concept is an analogy, conjecture, or proposed model component?
- Does it avoid claiming equivalence with existing physics unless that equivalence is demonstrated?
- Are external scientific references or inspiration sources kept close to the claims that rely on them?

## Public Framing

- Does the page match the public positioning of Ontoverse as a self-developed conceptual framework?
- Does it avoid mystical framing, unsupported certainty, and grand-theory language?
- Does it preserve the distinction between useful speculation and unsupported assertion?
- Does it make the project easier to criticize rather than harder to challenge?

## Promotion Rule

Move material from `notes/` into `framework/`, `models/`, `glossary/`, or public-facing pages only when the terminology and boundaries are stable enough to be reused.

When in doubt, keep the material in `notes/` and mark it as exploratory.
